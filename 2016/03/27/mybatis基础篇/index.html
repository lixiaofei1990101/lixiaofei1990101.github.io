1.Mybatis框架介绍

mybatis是一个持久层的框架，是apache下的顶级项目。
Mybatis托管在：https://github.com/mybatis/mybatis-3/releases，下图是mybatis持久层框架和后台数据库交互的过程。

2.mybatis-config.xml文件详解

这个可以看就mybatis框架的基础配置文件。也是后续和spring整合的桥梁。它内部的节点主要也就下面这样的一种结构。
注意点：
mybatis在加载文件的时候properties属性将会被优先获取，所以我们在给properties文件配置键值的时候一定要注意它的特殊性，不然在我们mappers的mapper.xml文件中如果和这里冲突了，那么mybatis获取的值将会获取到properties中的。
properties（属性）
settings（全局配置参数）
typeAliases（类型别名）
typeHandlers（类型处理器）
objectFactory（对象工厂）
plugins（插件）
environments（环境集合属性对象）
environment（环境子属性对象）
transactionManager（事务管理）
dataSource（数据源）
mappers（映射器）

2.1 properties节点

一般是用来加载classpath路径下面的比如说db.properties文件的.例： 

<properties resource="db.properties"></properties>

2.2 全局参数配置setting节点

<settings>
    <!-- 打开延迟加载 的开关 -->

</settings>

Mappers节点主要是用来加载mapper.xml文件的，同样的和别名的定义一样，它也有多种的加载方式.

2.5.1 Mybatis加载mapper.xml文件

<mappers>
    1、通过resource方法一次加载一个映射文件
    <!-- <mapper resource="mapper/UserMapper.xml"/> -->
    2、通过mapper接口加载单个 映射文件遵循一些规范：需要将mapper接口类名和mapper.xml映射文件名称保持一致，且在一个目录 中上边规范的前提是：使用的是mapper代理方法
    <mapper class="cn.itcast.mybatis.mapper.UserMapper"/>
    3、批量加载mapper指定mapper接口的包名，mybatis自动扫描包下边所有mapper接口进行加载遵循一些规范：需要将mapper接口类名和mapper.xml映射文件名称保持一致，且在一个目录中上边规范的前提是：使用的是mapper代理方法
    <package name="cn.itcast.mybatis.mapper"/>
</mappers>

2.5.2 Spring整合myabtis后加载mapper.xml文件

<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">

    <property name="mapperLocations">
        <list>
            <value>classpath:mapper/*.xml</value>
        </list>
    </property>
</bean>

3.Mybatis持久层的开发

通过上面，我们知道了mybatis就是一个持久层的框架，但mybatis仍然属于一种半自动的框架，在实际的业务中还是需要我们去编写和业务层交互的dao层，那么dao层又是怎么去开发的呢？总的来说一般是分成了两种方式：1、原始dao层的开发。2、mapper代理的方式。下面就具体分析下两种方式的区别。

3.1 原始dao层的开发

原始的dao层开发，程序员需要自己编写dao的接口和实现类，需要向dao实现类中注入SqlSessionFactory，
在方法体内通过SqlSessionFactory创建SqlSession.然后调用SqlSession的方法和数据库交互.
1.接口定义

public interface UserDao {
    public User findUserById(int id) throws Exception;
}

2.实现接口

public class UserDaoImpl implements UserDao {
    private SqlSessionFactory sqlSessionFactory;
        //构造方法注入SqlSessionFactory
        public UserDaoImpl(SqlSessionFactory sqlSessionFactory) {
        this.sqlSessionFactory = sqlSessionFactory;
    }
    @Override
    public User findUserById(int id) throws Exception {
        SqlSession sqlSession = sqlSessionFactory.openSession();
        User user = sqlSession.selectOne("test.findUserById", id);
        sqlSession.close();
        return user;
    }
}

3.编写测试类

public class UserDaoImplTest {
    private SqlSessionFactory sqlSessionFactory;
    // 此方法是在执行testFindUserById之前执行
    @Before
    public void setUp() throws Exception {
        // 加载mybatis基础配置文件
        String resource = "mybatis-config.xml";
        // 得到配置文件流
        InputStream inputStream = Resources.getResourceAsStream(resource);
        // 创建会话工厂，传入mybatis的配置文件信息
        sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
    }
    @Test
    public void testFindUserById() throws Exception {
        // 创建UserDao的对象，调用构造方法，传递SqlSessionFactory对象
        UserDao userDao = new UserDaoImpl(sqlSessionFactory);
        // 调用UserDao的方法
        User user = userDao.findUserById(1);
        System.out.println(user);
    }
}

到这里，整个的流程基本就算是完成了，可以看到相对来说，采用dao层的原始开发方法还是有很多的弊端的。
1、sqlSession.selectOne（“statementId”,Object）,我们在调用sqlSession的方法的时候对statementId进行了硬编码。
2、在调用方法的时候我们传入的参数是哥Object对象，这样的话，在编译的时候不会出现报错的情况，但是也许我们传入的参数并不是我们业务所需要的参数。
注：如果一定要采用这种原始的方法话，在和spring整合以后建议是优先编写一个工具类，去实现SqlSession的接口，或者继承SqlSessionTemplate复写父类的方法，然后去指定传入的参数类型也可以。

3.2 Mapper代理的方式

这应该也算是mybatis比较亮的一个地方把，采用这种方式，mybatis会自动生成接口的实现类。但必须遵循一定的开发规范。
1、在我们开发的mapper.xml中的namespace的值必须是mapper的接口地址,例:

<mapper namespace="com.lixf.test.TestDao">

2、接口中的定义方法名必须和mapper.xml的statementId保持一致
3、接口中的传入参数类型（比如说查询中的查询条件参数类型）必须和mapper.xml的parameterType保持一致。
4、同样的接口中的返回数据类型也必须和mapper.xml中的resultType保持一致
以上这四点是采用mapper代理方式开发持久层必须遵循的开发规范。

4.mapper.xml分析

这部分主要是用来存放持久层和DB进行交互的sql语句 部分，也是mybatis的关键部分。至于sql语句的学习，这里就不多写了，这部分主要分为：
Mapper.xml各个节点的分析。
xml的增删改查基本语句输入和输出参数的类型。
搞基映射resultMap返回类型的分析，和查询对应关系（复杂）。
动态的sql语句（复杂）。

4.1 Mapper.xml节点总览（常用）

4.1.1 数据库的新增操作（parameterType:新增输入参数，第二个和第三个参数配合使用用于生成自动主键的）例:

1、<insert id="" parameterType="" useGeneratedKeys="true" keyProperty="id"></insert>
2、自增长生成key插入
<insert id="insertBlog" parameterType="Blog">
    <selectKey keyProperty="studentID" resultType="String" order="BEFORE">
        select nextval('id')
    </selectKey>
    insert into Blog
    (id,title,author_id)
    values
    (#{id},#{title},#{authorId})
</insert>

4.1.2 数据库的删除操作（parameterType:删除输入参数）例:

<delete id="deleteUser" parameterType="int">
    delete from user where id=#{id}
</delete>

4.1.3数据库的修改操作（parameterType:修改输入参数）例:

<update id="updateUser" parameterType="User" >
    Update user set
    userName=#{userName},userAge=#{userAge},userAddress=#{userAddress}     where id=#{id}
</update>

4.1.4 数据库的查询操作（parameterType:查询的输入参数，resultType查询返回的结果类型）例:

<select id="selectUsersByName" parameterType="string" resultType="User">
    select * from user where user.userName = #{userName}
</select>

4.1.5 主要是用来定义sql片段的，增加sql的复用性（id是sql片段的唯一标识）例:

<sql id="blog_column">id,title,author_id as authorId</sql> 
<select id="selectBlog_as_map" parameterType="int" resultType="hashmap">  
   select <include refid="blog_column"/> from Blog where id = #{id}  
</select>

4.1.6 resultMap主要用来描述如何从数据库结果集中来加载你的对象 例:

<resultMap type="cn.itcast.mybatis.po.Orders" id="OrdersUserResultMap">
    <id column="id" property="id"/>
    <result column="user_id" property="userId"/>
    <result column="number" property="number"/>
    <result column="createtime" property="createtime"/>
    <result column="note" property="note"/>
</resultMap>

4.2 高级映射resultMap

在resultMap节点下面主要有以下几个节点:Id,Result,Association,Collection。

4.2.1 一对一的关系（一个订单一个用户）

Type：resultMap对应的实体，id作为查询Result返回类型的值
<resultMap type="cn.itcast.mybatis.po.Orders" id="OrdersUserResultMap">
    Id：查询列表中的唯一表示，如果是多少列组成唯一表示，则需要配置多个id。
    Property：实体对象中对应的属性值。
    Column：数据库的字段值。
    <id column="id" property="id"/>
    <result column="user_id" property="userId"/>
    <result column="number" property="number"/>
    <result column="createtime" property="createtime"/>
    <result column="note" property="note"/>
    Association:用于映射关联查询单个对象的信息
    Property:返回主体类型中的Object对象属性
    javaType:返回主体对象中的Object对象的路径
    <association property="user"  javaType="cn.itcast.mybatis.po.User">
        <id column="user_id" property="id"/>
        <result column="username" property="username"/>
        <result column="sex" property="sex"/>
        <result column="address" property="address"/>
    </association>
</resultMap>

4.2.2 一对多的关系（一个订单对应多个订单明细）

继承性：其它的和上面一样，但其中的extends却体现了mybatis的resultMap的继承性，这里就相当于把上面resultMap中间的部分加入到了下面这个resultMap中间。
<resultMap type="cn.itcast.mybatis.po.Orders" id="OrdersAndOrderDetailResultMap" extends=
"OrdersUserResultMap">
Collection：主要用于将查询的多条记录映射到对象集合中。
Property:指明映射到返回类型中的那个属性。
ofType:指明List集合属性的类型。
<collection property="orderdetails" ofType="cn.itcast.mybatis.po.Orderdetail">
         <id column="orderdetail_id" property="id"/>
         <result column="items_id" property="itemsId"/>
         <result column="items_num" property="itemsNum"/>
         <result column="orders_id" property="ordersId"/>
    </collection>
</resultMap>

4.2.3 多对多的关系（用户购买的商品信息查询）

4.3 mybatis的动态sql

MyBatis的动态SQL是基于OGNL表达式的，它可以帮助我们方便的在SQL语句中实现某些逻辑。
MyBatis中用于实现动态SQL的元素主要有：if,choose(when,otherwise),trim,where,set,foreach

4.3.1 if元素

if元素就是简单的条件判断，利用if语句我们可以实现某些简单的条件选择。例

<select id="getUser" parameterType="User" resultType="User">
    select * from user where 1 = 1
    <if test="username != null and username != ''">
        and title = #{title}
    </if>
</select>

4.3.2 choose元素

choose在mybatis中的用法其实和jstl标签中的用法差不多的.例:

<select id="getOrder" parameterType="Order" resultType="Order">
    select * from order where 1 = 1
    <choose>
        <when test="orderNo != null and orderNo != ''">
            orderNo = #{orderNo}
        </when>
        <otherwise>
            <if test="createDate != null and createDate != ''">
                createDate = #{createDate}
            </if>
        </otherwise>
    </choose>
</select>

4.3.3 where元素

where元素的作用主要是简化SQL语句中where中的条件判断的使用这个元素以后，mybatis会只能的把首个and或者or给忽略。例:

<select id="getOrder" parameterType="Order" resultType="Order">
    select * from order
    <where>
        <if test="orderNo != null and orderNo != ''">
            orderNo = #{orderNo}
        </if>
    </where>
</select>

4.3.4 trim元素.

trim元素为格式化元素.可以与其他标识完成where和set的功能Prefix：前缀增加suffix：后缀增加prefixOverrides：自动判断前置suffixOverrides： 自动判断后置。
无疑，在mybatis这是一个非常强大的元素，下面我们就来看看一些相关的例子。例:

1、trim替代set
<update id="updateOrder" parameterType="Order">
    update order
    <trim prefix="set" suffixOverrides="," suffix="where orderNo = #{orderNo}">
        <if test="createDate != null and createDate != ''">
            createDate = #{createDate}
        </if>
    </trim>
</update>
2、trim替代where
<select id="getOrder" parameterType="Order" resultType="Order">
    select * from order
    <trim prefix="where" prefixOverrides="and|or">
        <if test="orderNo != null and orderNo != ''">
            orderNo = #{orderNo}
        </if>
        <if test="createDate != null and createDate != ''">
            and createDate = #{createDate}
        </if>
    </trim>
</select>
3、insert语句中trim的应用
<insert id="addOrder" parameterType="Order">
    insert into order(
        <trim suffix="" suffixOverrides=",">
            <if test="orderNo != null and orderNo != ''">
                ORDERNO,
            </if>
            <if test="createDate != null and createDate != ''">
                CREATEDATE,
            </if>
        </trim>
    )values(
        <trim suffix="" suffixOverrides=",">
            <if test="orderNo != null and orderNo != ''">
                #{orderNo},
            </if>
            <if test="createDate != null and createDate != ''">
                #{createDate},
            </if>
        </trim>
    )
</insert>

4.3.5 set元素

set一般用于数据更新的修改操作。例:

<update id="updateOrder" parameterType="Order">
    update order
    <set>
        <if test="createDate != null and createDate != ''">
            CREATEDATE = #{createDate}
        </if>
    </set>
    where ORDERNO = #{orderNo}
</update>

4.3.6 foreach元素

foreach元素一般用户循环遍历的。例:

1、直接传入简单类型的list集合参数，这个时候collection必须是list
<select id="getOrder" parameterType="Order" resultType="Order">
    select * from order where ORDERNO in
    <foreach collection="list" item="item" open="(" close=")" separator=",">
        #{item}
    </foreach>
</select>
2、传入list对象集合，同样的collection必须是list
<select id="getOrder" parameterType="Order" resultType="Order">
    select * from order where ORDERNO in
    <foreach collection="list" item="item" open="(" close=")" separator=",">
        #{item.orderNo}
    </foreach>
</select>
3、map的方式传入list集合，这里的list，collection类型不限定
<select id="getOrder" parameterType="map" resultType="Order">
    select * from order where ORDERNO in
    <foreach collection="orderNos" item="item" open="(" close=")" separator=",">
        #{item}
    </foreach>
</select>

4.4 mybatis输入输出参数

前面我们已经把mybatis自带的一些简单参数类型列举出来了，这里就不重复列举了。下面我们主要针对自定义的一些参数类型进行分析

<select id="" parameterType="User" parameterMap=""></select>
<select id="" resultType="User" resultMap="resultMap" ></select>

一般情况下，除了parameterMap这个以外，其他在mybatis里面都是经常用的。无论是输入或者输出的参数类型我们除了可以引用myabtis自带的返回参数类型外，还可以自定义一些参数类型，常见的就是对应数据库表的实体对象和Map，但是往往比如说我们在做查询的时候，会涉及到多表的查询，可能需要用到另外一个实体中的某个属性，这个时候，我们就可以采用POJO对输入和输出的参数进行包装。可以继承某个实体，然后再另外定义一些属性，用来作为查询条件的属性。
