github上传项目

2016-9-12 周一 09:28

Github介绍

Github是一个代码托管平台和开发者社区，开发者可以在Github上创建自己的开源项目并与其他开发者协作编码。开源项目可以免费托管，私有项目需付费。官方网址是:https://github.com/。一开始的时候，自己编写的东西一直苦恼没办法在公司和家里同步，从今年的四五月份为了搭建自己的个人博客才开始接触github。下面就我本地项目如果推送到github稍微做下介绍。
首先下载git版本控制器，并安装，下载路径:https://git-for-windows.github.io/我这里不多介绍，百度一下很多。然后到官方网站注册一个自己的Github账号。然后新建一个项目，这里也就不做过多的演示了。账号注册号了，项目建好后。下面就是具体的操作过程了。

步骤

1、生成本地公钥，git配置。

一旦我们的git安装好了以后，你点击右键会出现Git base here这个选项，然后在里面输入 : ssh-keygen -t rsa -C “your_email@youremail.com”来生成推送到github的ssh协议公钥，一般这个公钥是生成在用户的.ssh目录下面，例如：用户/testuser/.ssh/id_rsa.pub 。这个文件下面的内容就是你本地的公钥了，然后复制里面的内容到个人账号下。登陆以后，点击个人的那个小三角形，下面有一个settings选项，点击进去以后找到SSH and GPG keys选项，然后New SSH key，title随便填写一个，下面大框里面就是咱们刚刚在id_rsa.pub里面复制出来的内容了。OK，到这里就算是完成了我们本地到github链接的桥梁了，下面再输入:ssh -T git@github.com,测试一下。连接成功以后，再设置下我们的用户名和邮箱，因为每次提交的时候，都会记录这些东西。

2、推送本地项目。

在推送项目之前，我们先在我们的账号里面建立一个仓库，创建好以后，会生成一个.git的路径，这个就是我们后续推送和下载的路径了。这里也省略了，很简单的操作，推送我们可以使用git直接推送，也可以安装一个git的版本控制器进行推送，为了方便一般采用TortoiseGit这个工具去对版本进行控制，图形化界面有点类似SVN。感觉这个用起来比较的顺手。当然汉化一下就更加好咯。相信SVN大家都会使用咯。就不详细介绍。
