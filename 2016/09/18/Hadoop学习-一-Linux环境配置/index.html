Hadoop学习(一) Linux环境配置

2016-9-18 周日 15:11

个人一般都不会有多台电脑给你去搭建什么集群，只能选择用虚拟机了。我自己用的是VMware来安装的centos7的linux系统具体的步骤，可以去网上参考一下，不是很复杂，稍微细心一点，很容易就安装好了。下面就直接说说在学习hadoop中linux系统一些基本的配置。

关闭linux防火墙

禁用系统默认防火墙

因为我电脑安装的是centos7，而这个版本默认的防火墙是firewall。首先我们要做的是禁止掉centos7的默认防火墙
1、systemctl stop firewalld.service 停止firewall。
2、systemctl disable firewalld.service 禁止firewall开机启动。

安装iptable防火墙

yum install -y iptable，如下图所示表示已经下载好了iptable防火墙的安装包，接下来就是安装了。
yum install iptables-services 执行iptable的安装如下，一路逮下去，直接Y。
完成iptable的安装以后，进入到/etc/sysconfig/下面可以看到一个iptables的文件，这里面就是配置的防火墙一些进站还有端口的限制，简单的可以参考下面的一些设置，来编辑iptables，然后保存就OK了。

防火墙设置规则

#查看iptables现有规则
iptables -L -n
#先允许所有,不然有可能会杯具
iptables -P INPUT ACCEPT
#清空所有默认规则
iptables -F
#清空所有自定义规则
iptables -X
#所有计数器归0
iptables -Z
#允许来自于lo接口的数据包(本地访问)
iptables -A INPUT -i lo -j ACCEPT
#开放22端口
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
#开放21端口(FTP)
iptables -A INPUT -p tcp --dport 21 -j ACCEPT
#开放80端口(HTTP)
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
#开放443端口(HTTPS)
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
#允许ping
iptables -A INPUT -p icmp --icmp-type 8 -j ACCEPT
#允许接受本机请求之后的返回数据 RELATED,是为FTP设置的
iptables -A INPUT -m state --state  RELATED,ESTABLISHED -j ACCEPT
#其他入站一律丢弃
iptables -P INPUT DROP
#所有出站一律绿灯
iptables -P OUTPUT ACCEPT
#所有转发一律丢弃
iptables -P FORWARD DROP
#如果要添加内网ip信任（接受其所有TCP请求）
iptables -A INPUT -p tcp -s 45.96.174.68 -j ACCEPT
#过滤所有非以上规则的请求
iptables -P INPUT DROP
#要封停一个IP，使用下面这条命令：
iptables -I INPUT -s ***.***.***.*** -j DROP
#要解封一个IP，使用下面这条命令:
iptables -D INPUT -s ***.***.***.*** -j DROP

iptable防火墙的常用操作

#注册iptables服务
#相当于以前的chkconfig iptables on
systemctl  enable  iptables.service
#开启服务
systemctl  start  iptables.service
#查看状态
systemctl  status  iptables.service
#禁用防火墙启动
Systemctl  disable  iptables.service

OK！到这里我相信防火墙应该是被你关闭了，当然如果你使用的是比如6.4等其他低版本的，操作起来就方便很多了，直接就是service iptables status 等等。这个地方不知道的可以百度一下，有很多相关的介绍，这里我就不一一介绍了。OK！咱们接着往下看。

网络的设置

在我们的虚拟机安装完成以后，我们打开我们的网络设置可以看到多出了两块网卡。

网络连接

在你电脑可以连接网络的时候，采用NAT模式，共享主机的IP地址。
系统安装默认就是这种模式的，然后我们在linux系统中查看IP地址（192.168.222.129），这里走的是主机连接网络的公网IP地址。接着在windows的dos操作平台里面ping到IP地址，可以看到网络已经共享。
2.2.2、无网络的情况，则采用仅主机模式，这样就能在专用网络内连接虚拟机了 ，这种情况下，不需要我们的电脑连接网络，虚拟机就能和主机之间建立网络连接。但是没法上网，因为你建立的是虚拟机和主机之间的连接。
这个时候，我们再从新启动虚拟机，查看IP地址，变成了（192.168.77.128），其中这里走的是我们windows的IP地址。咱们ping一下，可以看到是正常的。其实说白了就是局域网内计算机之间实现网络的通讯。为了方便，在接下来的学习中都采用第一种方式建立连接（注意：在我们使用这种方式的时候，一定要启用对应的网卡）

这里是通过桌面的形式来修改我们的IP地址，当然也可以使用命令的方式，进入到我们/etc/sysconfig/network-scripts目录下面可以看到一个ifcfg-eno16777736文件，这个文件就是网络的配置文件了，可以通过修改里面的内容来编辑网络。OK！到这里就算是完成了对静态IP地址的设置了

修改主机名

hostnamectl set-hostname <自定义的主机名>

配置域名映射

配置主机的IP地址和主机名的映射，其实就是配置本地的DNS，这样通过主机名就能访问到我们的主机了，不需要去记对应的IP地址。修改/etc/hosts文件，将IP和主机名对应就好了。
