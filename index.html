Hadoop学习(五)-MapReduce架构原理

2016-10-25 周二 11:11

概述

hadoop主要是用于应对海量数据的存储和计算的，前面hdfs文件系统，则重点是用于海量数据的存储。那么有了这么多数据，我们又该怎么在这些数据里面来提取我们需要的信息呢？这个时候hadoop中出现了一个非常重要的成员MapReduce。
MapReduce是一种编程模型，用于大规模数据集(大于1TB)的并行运算。注意这个并行运算，它不再是我们传统意义上的串行计算，它可以在不同的进程中进行并行的运算。概念”Map(映射)”和”Reduce(归约)”，是它们的主要思想，都是从函数式编程语言里借来的，还有从矢量编程语言里借来的特性。它极大地方便了编程人员在不会分布式并行编程的情况下，将自己的程序运行在分布式系统上。 当前的软件实现是指定一个Map(映射)函数，用来把一组键值对映射成一组新的键值对，指定并发的Reduce(归约)函数，用来保证所有映射的键值对中的每一个共享相同的键组。

NameNode

工作范畴

1.1、metadata用来存储元数据元数据的格式：文件名、备份数，切分的块，所在的机器等等一些文件描述性信息构成的保存在内存中的，这样查找起来就会快很多。样例:(web.log,3{blk_1,blk_2},[{blk_1:[node1,node2,node3]},{blk_2:[{node1,node3,node4}]}])。这也是客户端请求读取数据NameNode去查找DataNode的依据。

1.3、fsimage是元数据镜像文件（保存文件系统的目录树），存储某一时段namenode内存元数据。（hdfs-site.xml的dfs.name.dir属性）。
1.4、edits是元数据操作日志
(记录每次保存fsimage之后到下次保存之间的所有hdfs操作)。
1.5、fstime 保存最近一次活跃点（checkpoint）的时间

Hadoop学习(三) 伪分布式搭建

2016-9-20 周二 16:32

Hadoop运行环境安装

Hadoop是用java语言编写的，要运行Hadoop，java环境是必须的。其实也就是安装JDK和相关环境变量的配置。

  • JDK的安装
    首先查看系统自带的java相关的程序，全部卸载掉。
    rpm -qa|grep java（查看）
    yum -y remove java程序（卸载）
    然后下载一个jdk-7u76-linux-x64.tar的JDK，然后将这个压缩包放到我们的linux上面，采用tar –zxvf filename 方式解压文件，然后用mv的命令将我们解压的目录放到指定的目录下面，剩下的就是配置环境变量了，首先找到/etc/profile这个文件到最后面加入环境变量。
    export JAVA_HOME=/opt/software/jdk1.7.0_76
export PATH=$PATH:$JAVA_HOME/bin
    然后执行这个文件source /etc/profile。
    这样就算是完成了linux的java环境的搭建。

Hadoop学习(二) Hadoop生态圈

2016-9-19 周一 11:07

Hadoop概况

关于时下最热的技术潮流，无疑大数据是首当其中最热的一个技术点，关于大数据的概念和方法论铺天盖地的到处宣扬。关于大数据项目可以真正被落地实施的解决方案就是：Hadoop为核心的的一些列开源分布式解决方案。
Hadoop是一个由Apache基金会所开发的分布式系统基础架构。
用户可以在不了解分布式底层细节的情况下，开发分布式程序。充分利用集群的威力进行高速运算和存储。
Hadoop实现了一个分布式文件系统(Hadoop Distributed File System)，简称HDFS。HDFS有高容错性的特点，并且设计用来部署在低廉的(low-cost)硬件上;而且它提供高吞吐量(high throughput)来访问应用程序的数据，适合那些有着超大数据集(large data set)的应用程序。HDFS放宽了(relax)POSIX的要求，可以以流的形式访问(streaming access)文件系统中的数据。
Hadoop的框架最核心的设计就是:HDFS、MapReduce和Yarn。HDFS为海量的数据提供了存储，则MapReduce为海量的数据提供了计算。Yran则可以看成一个调度系统。
